## On-Page SEO Requirements

### Page Title (Title Tag)
- 50-60 characters maximum
- Primary keyword near the beginning
- Include brand name at end: "| Talosix"
- Format: "[Primary Keyword]: [Value Proposition] | Talosix"
- Example: "Clinical Trial EDC Software: Faster Studies, Better Data | Talosix"

### Meta Description
- 150-160 characters
- Include primary keyword naturally
- Include a clear value proposition and implicit CTA
- Example: "Talosix EDC helps clinical teams build studies in days, not weeks. Reduce query rates by 65% with intelligent data capture. Request a demo."

### URL Structure
- Short, readable, keyword-inclusive
- Use hyphens, lowercase
- Pattern: `/blog/[keyword-slug]` or `/resources/[keyword-slug]`
- Example: `/blog/ecrf-design-best-practices`

### Heading Structure
- **H1:** One per page. Contains primary keyword. Matches or closely mirrors page title.
- **H2:** Section headings. Include secondary keywords naturally. 3-6 per article.
- **H3:** Subsection headings under H2s. Use for scanability and related terms.
- **Do not skip levels** (H1 > H3 without H2).

### Content Body
- Primary keyword in first 100 words
- Keyword density: 1-2% for primary keyword (natural, not forced)
- Secondary keywords distributed through body and headings
- Semantic variations and related terms throughout (LSI keywords)
- Minimum word count by type:
  - Blog post: 1,000-1,500 words
  - Pillar page: 2,500-4,000 words
  - Product page: 500-1,000 words
  - Glossary entry: 300-500 words

### Images and Media
- Descriptive alt text including keywords where relevant
- File names: `clinical-trial-edc-dashboard.png` not `screenshot1.png`
- Compressed for page speed (<200KB per image)
- Captions where helpful for context

### Internal Linking Strategy

**Link architecture:**
- Every blog post links to 2-3 related blog posts
- Every blog post links to 1 relevant product or feature page
- Product pages link to supporting blog posts and resources
- Pillar pages link extensively to cluster content and vice versa

**Anchor text:**
- Use descriptive, keyword-relevant anchor text
- Vary anchor text (avoid exact-match keyword for every link)
- Example: Link "learn more about eCRF design best practices" to the eCRF best practices post

**Topic cluster model:**
```
Pillar: "Complete Guide to Clinical Trial EDC"
  |-- Cluster: "eCRF Design Best Practices"
  |-- Cluster: "EDC System Validation Guide"
  |-- Cluster: "CDISC CDASH Compliance in EDC"
  |-- Cluster: "Query Management Optimization"
  |-- Cluster: "EDC for Decentralized Trials"
  |-- Cluster: "21 CFR Part 11 EDC Requirements"
```

## Technical SEO Considerations

- **Schema markup:** Use Article schema for blog posts, Product schema for product pages, FAQ schema for FAQ sections
- **Canonical tags:** Set on all pages to prevent duplicate content
- **Open Graph tags:** Title, description, image for social sharing
- **Page speed:** Target <3 second load time; compress images, minimize scripts
- **Mobile-first:** All content must render properly on mobile devices
- **XML sitemap:** All published content pages included
- **Robots.txt:** Ensure content pages are crawlable

## Content Refresh Cycle

- **Quarterly:** Review top 20 pages by traffic. Update statistics, add new sections, refresh internal links.
- **On regulatory change:** Update all affected content within 2 weeks of final guidance publication (ICH, FDA, EMA).
- **Annual:** Full content audit. Identify underperforming pages for consolidation, update, or removal.
- **Update signals:** Add "Last updated: [date]" to all evergreen content. Search engines favor freshness.

3. Schema markup recommendations
4. 3-5 internal linking suggestions with anchor text
5. Social media snippets for content promotion
6. Recommended content refresh date
