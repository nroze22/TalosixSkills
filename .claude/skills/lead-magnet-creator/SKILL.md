# Lead Magnet Creator for Talosix EDC

## Purpose

Design and produce lead magnets that clinical trial professionals actually want -- valuable enough to exchange their email address for, specific enough to attract qualified prospects, and strategic enough to feed a nurture sequence that leads to demo requests. This skill covers the full lifecycle: concept development, content creation, landing page copy, email gate strategy, and post-download nurture.

## Lead Magnet Strategy Framework

### What Makes a Great Lead Magnet for Clinical Trial Buyers

Clinical trial professionals are skeptical of gated content. They work in a regulated industry where precision matters. Generic eBooks do not work. Effective lead magnets for this audience must be:

1. **Immediately actionable** -- They can use it today, not "someday"
2. **Specific** -- "21 CFR Part 11 Compliance Checklist" beats "Digital Transformation in Healthcare"
3. **Operationally relevant** -- Connected to their actual daily workflows
4. **Credible** -- Reflects genuine industry expertise, not repackaged blog posts
5. **Time-respectful** -- Can be consumed in 10-15 minutes or used as a reference tool

### Lead Magnet Quality Test

Before finalizing any lead magnet concept, it must pass all five:
- [ ] Would a Clinical Data Manager bookmark this for repeated use?
- [ ] Would a VP of Clinical Ops forward this to their team?
- [ ] Does it solve a problem that exists independently of Talosix's product?
- [ ] Is the value obvious from the title alone?
- [ ] Would someone regret NOT downloading it?

## Lead Magnet Types and Templates

### Type 1: Checklist

**Best for:** Compliance processes, vendor evaluations, audit preparation, system requirements.

**Structure:**
- Title: "[Number] [Topic] Checklist for [Audience]"
- Introduction: 2-3 paragraphs on why this checklist matters (max 150 words)
- Checklist sections: 4-6 categories with 5-8 items each
- Each item: Checkbox + item description + brief guidance note (1-2 sentences)
- Footer: About Talosix (2 sentences) + CTA

**Proven checklist concepts for clinical trials:**

1. **21 CFR Part 11 Compliance Checklist**
   - Sections: Audit trails, electronic signatures, access controls, data integrity, system validation, documentation
   - Audience: IT, Validation, Data Management
   - Funnel position: Top-of-funnel (broad regulatory interest)

2. **EDC Vendor Evaluation Scorecard**
   - Sections: Core functionality, compliance, integrations, implementation, support, pricing, scalability
   - Format: Scored rubric (1-5 scale) with weighted categories
   - Audience: Clinical Ops, IT, Procurement
   - Funnel position: Mid-funnel (active evaluation)

3. **Study Build Readiness Checklist**
   - Sections: Protocol review, eCRF design requirements, edit check specifications, data flow mapping, UAT planning, site readiness
   - Audience: Data Management, Clinical Ops
   - Funnel position: Top-of-funnel (operational improvement)

4. **Database Lock Preparation Checklist**
   - Sections: Query resolution, data cleaning, medical review, SAE reconciliation, coding completion, audit trail review, lock procedures
   - Audience: Data Management
   - Funnel position: Top-of-funnel (operational improvement)

5. **EDC Migration Planning Checklist**
   - Sections: Data extraction, mapping, validation, testing, regulatory notification, site training, parallel run planning
   - Audience: IT, Data Management, Clinical Ops
   - Funnel position: Mid-funnel (considering platform change)

### Type 2: Template

**Best for:** Documents, specifications, evaluation frameworks that teams need to create regularly.

**Structure:**
- Title: "[Topic] Template for [Audience]"
- Introduction: When to use this template and how to customize it (100 words)
- Template body: Pre-filled with example content and placeholder instructions
- Customization guide: Section-by-section guidance on adapting to their needs
- Footer: About Talosix + CTA

**Proven template concepts:**

1. **EDC User Requirements Specification (URS) Template**
   - Sections: Functional requirements, regulatory requirements, technical requirements, integration requirements, validation requirements
   - Pre-filled with common EDC requirements that teams can customize
   - Audience: IT, Validation
   - Funnel position: Mid-funnel

2. **eCRF Design Specification Template**
   - Sections: Form inventory, field definitions, edit check specifications, data flow, coding requirements
   - Audience: Data Management
   - Funnel position: Top-of-funnel

3. **EDC Validation Plan Template**
   - Sections: Scope, approach, test strategy, acceptance criteria, roles, timeline, documentation requirements
   - Audience: IT, Validation
   - Funnel position: Mid-funnel

### Type 3: Mini-Guide / Playbook

**Best for:** Educational content on a specific topic, methodology explanations, best practice collections.

**Structure:**
- Title: "The [Audience]'s Guide to [Topic]" or "[Topic]: A [Number]-Step Playbook"
- Length: 8-15 pages (2,500-5,000 words)
- Structure: Introduction, 4-6 chapters of 400-800 words each, conclusion with next steps
- Design: Professional but not design-heavy -- text-led with callout boxes, pull quotes, and simple charts
- Footer: About Talosix + CTA

**Proven guide concepts:**

1. **Clinical Trial Data Management Best Practices Guide**
   - Topics: Data quality metrics, edit check strategy, query management optimization, CDISC compliance, database lock acceleration
   - Audience: Data Management
   - Funnel position: Top-of-funnel

2. **The Clinical Ops Director's Guide to EDC Selection**
   - Topics: When to switch EDC, evaluation criteria, build vs. buy decisions, implementation planning, ROI calculation
   - Audience: Clinical Ops (decision makers)
   - Funnel position: Mid-funnel

3. **Decentralized Trial Data Capture: What You Need to Know**
   - Topics: ePRO integration, eConsent, remote monitoring, data reconciliation, regulatory considerations
   - Audience: Clinical Ops, Data Management
   - Funnel position: Top-of-funnel

### Type 4: Assessment / Calculator Tool

**Best for:** Interactive tools that provide personalized outputs, self-assessments, benchmarking.

**Structure:**
- Title: "[Topic] Calculator" or "[Topic] Assessment"
- Format: Series of questions or inputs that produce a personalized result
- Provide the content, logic, and scoring -- implementation as interactive web tool is a separate step
- Include a static PDF version for those who prefer not to use interactive tools

**Proven assessment concepts:**

1. **Study Build Time Calculator**
   - Inputs: Number of forms, complexity level, edit check count, amendment frequency, team size
   - Output: Estimated study build time with industry benchmark comparison
   - Shows where time savings are possible
   - Audience: Data Management, Clinical Ops

2. **EDC Readiness Assessment**
   - Inputs: 15-20 questions about current EDC capabilities, pain points, team structure
   - Output: Readiness score across 5 dimensions with recommendations
   - Audience: Clinical Ops, IT

3. **Data Quality Benchmarking Tool**
   - Inputs: Query rate, resolution time, error rate, clean data percentage
   - Output: Comparison against industry benchmarks with improvement opportunities
   - Audience: Data Management

### Type 5: Benchmark Report

**Best for:** Original research or curated data that the audience cannot easily find elsewhere.

**Structure:**
- Title: "[Year] [Topic] Benchmark Report"
- Length: 10-20 pages
- Data: Original survey data, aggregated customer data (anonymized), or curated industry data
- Key findings with charts and commentary
- Implications for the reader's organization
- About Talosix + methodology notes

**Proven benchmark concepts:**

1. **Clinical Trial EDC Performance Benchmarks**
   - Metrics: Study build time, query rates, database lock cycles, site adoption rates
   - Segmented by trial phase, therapeutic area, organization size
   - Audience: All clinical trial personas

2. **Clinical Data Management Salary and Staffing Survey**
   - Metrics: Staffing ratios, salaries, tool usage, biggest challenges
   - Audience: Data Management leadership
   - High sharing potential

## Landing Page Copy

For every lead magnet, produce landing page copy following this structure:

### Landing Page Template

```
H1: [Benefit-focused title that includes the lead magnet name]
Subhead: [One sentence on what they get and why it matters]

[HERO SECTION]
- Mockup or preview image of the lead magnet
- 3-4 bullet points on what is inside
- Form: Name, email, company, role (keep fields minimal)
- CTA button: "Download Now" / "Get the Checklist" / "Access the Guide"

[PROBLEM SECTION]
H2: [The problem this lead magnet solves]
- 2-3 sentences making the reader nod
- Specific to their clinical trial role

[WHAT'S INSIDE]
H2: What You Will Get
- 4-6 bullet points, each starting with a verb
- Each describes a specific section or deliverable

[SOCIAL PROOF]
- "Downloaded by X clinical trial professionals"
- Or: Customer quote about the topic
- Or: Talosix expert credentials

[ABOUT]
- 2 sentences about Talosix (not a sales pitch -- establish credibility)

[FORM REPEAT]
- Repeat CTA and form for those who scroll
```

### Form Field Strategy

**Minimum viable form (highest conversion):**
- Email address
- First name

**Standard form (balance of conversion and qualification):**
- Email address
- First name
- Company name
- Job title (dropdown)

**Extended form (lower conversion, higher lead quality):**
- All standard fields plus:
- Number of active studies
- Current EDC platform
- Timeline for EDC evaluation

**Rule:** Use minimum viable for top-of-funnel assets. Use standard for mid-funnel. Use extended only for high-value assets (benchmark reports, ROI calculators).

## Post-Download Nurture Sequence

Every lead magnet connects to a 5-email nurture sequence that moves the prospect from content consumer to demo requester.

### Nurture Sequence Template

| Email | Timing | Purpose | Subject Line Pattern | CTA |
|-------|--------|---------|---------------------|-----|
| 1 | Day 0 | Deliver lead magnet + quick tip | "Your [asset] is ready" | Download |
| 2 | Day 3 | Related insight (adds value, no ask) | "[Insight related to the topic]" | Read blog post |
| 3 | Day 7 | Deeper dive on the topic (educational) | "The [X] most teams miss about [topic]" | Watch video or read guide |
| 4 | Day 14 | Social proof (case study or metric) | "How [company type] achieved [result]" | View case study |
| 5 | Day 21 | Direct offer | "[First name], want to see this in action?" | Request a demo |

**Nurture rules:**
- Email 1-3 should provide value without asking for anything beyond a content click
- Email 4 introduces social proof but is still educational
- Email 5 is the only explicitly sales-oriented email
- Every email has a one-click unsubscribe
- Exit the sequence if the prospect requests a demo at any point
- Tone throughout: helpful peer, not persistent salesperson

### Nurture Sequence Copy Outline

For each email in the sequence, provide:
- Subject line (with A/B variant)
- Preview text
- Body copy outline (key points, not full text -- the email-campaign skill handles full copy)
- CTA with button text
- Segmentation notes (should this email differ by role or company type?)

## Gating Strategy

Not all content should be gated. Use this framework:

### Gate (require email)
- Checklists and templates (high utility, worth the exchange)
- Assessment tools and calculators (personalized output justifies the gate)
- Benchmark reports (original data is genuinely scarce)
- Multi-chapter guides (significant length and depth)

### Do Not Gate (leave open)
- Blog posts (SEO value depends on accessibility)
- One-pagers (too short to justify a gate)
- Infographics (shareability is the goal)
- Product overviews (prospects need this info to evaluate)

### Soft Gate (optional email for enhanced version)
- Guides: Ungated summary version + gated full PDF with bonus content
- Calculators: Basic version ungated + detailed version with benchmarks gated
- Reports: Key findings ungated + full report with data tables gated

## Content Production Process

### Step 1: Concept Selection
- Identify the target audience segment and funnel stage
- Choose lead magnet type based on the framework above
- Validate concept with the Lead Magnet Quality Test

### Step 2: Research
- Use WebSearch to check what competitors offer on this topic
- Identify what is freely available (our asset must be better or different)
- Use PubMed MCP for clinical trial research supporting the content
- Check Talosix existing content for reusable material

### Step 3: Content Creation
- Load brand-voice skill as background context
- Write the complete lead magnet content
- Include design guidance (layout, visual treatment, formatting)

### Step 4: Landing Page and Gate
- Write landing page copy using the template above
- Determine gate level (full, soft, or ungated)
- Write email gate copy

### Step 5: Nurture Sequence
- Outline the 5-email post-download sequence
- Identify supporting content for each email (blog posts, case studies, videos)
- Define segmentation logic if applicable

## Output Deliverables

For every lead magnet request, provide:

1. **Concept recommendation** with justification (type, title, audience, funnel position)
2. **Complete lead magnet content** (full text, formatted for design handoff)
3. **Design guidance** (layout suggestions, visual treatment, page count estimate)
4. **Landing page copy** (complete, following the template)
5. **Form field recommendation** with justification
6. **Email gate copy** (delivery email)
7. **Nurture sequence outline** (5 emails with subject lines, purpose, and CTA)
8. **Gating recommendation** (gate, soft-gate, or ungated with reasoning)
9. **Promotion plan** (how to drive traffic to the landing page -- channels, messaging)
10. **Success metrics** (download target, conversion rate benchmark, nurture-to-demo rate)
