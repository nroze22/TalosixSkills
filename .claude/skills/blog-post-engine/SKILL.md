# Blog Post Engine for Talosix EDC

## Purpose

Produce publication-ready blog posts that rank for clinical trial EDC keywords, establish Talosix as a thought leader, and drive measurable organic traffic. Every post follows a research-first methodology that identifies content gaps, optimizes for search intent, and embeds E-E-A-T signals throughout.

## Phase 1: Research and Strategy

Before writing a single word, complete this research protocol.

### 1.1 Keyword Analysis

**Primary keyword identification:**
- Use WebSearch to analyze search volume and competition for the target topic
- Identify the primary keyword (the one search phrase this post should rank for)
- Map search intent: informational, navigational, commercial, or transactional
- Check current SERP results for the primary keyword -- what content already ranks?

**Keyword cluster:**
- Identify 3-5 secondary keywords (semantically related terms)
- Identify 5-10 long-tail variations
- Map each to a section of the planned post

**Talosix keyword priorities (reference list):**

Primary (high intent):
- clinical trial EDC software
- electronic data capture system
- eCRF design software
- clinical data management system
- EDC platform for clinical trials
- EDC vendor comparison

Secondary (informational):
- clinical trial data collection best practices
- eCRF design best practices
- EDC system validation requirements
- CDISC compliant EDC
- risk-based data management
- decentralized trial data capture

Long-tail:
- "how to reduce clinical trial data entry errors"
- "EDC system for decentralized clinical trials"
- "CDASH compliant eCRF templates"
- "clinical trial data quality metrics"
- "21 CFR Part 11 EDC requirements"
- "EDC vendor evaluation checklist"

### 1.2 Competitive Content Analysis

**Search the primary keyword and analyze the top 5 ranking results:**

For each competing piece, note:
- Word count and depth
- Structure (headings, sections, multimedia)
- Unique angle or data they present
- Gaps -- what do they miss that our audience would want?
- Quality signals -- author expertise, citations, freshness

**Content gap identification:**
- What questions does the audience have that competitors do not answer?
- What data or perspective can Talosix uniquely provide?
- Where can we go deeper or more specific?
- What format improvements would make our piece more useful?

**Competitive advantage angle:**
Write one sentence describing why Talosix's blog post will be the best result for this query. This becomes the north star for the writing phase.

### 1.3 Search Intent Mapping

Classify the primary keyword's search intent and match the post format:

| Intent | Post Format | Reader Wants |
|--------|------------|--------------|
| Informational | Educational guide, explainer | Learn something, understand a concept |
| Commercial investigation | Comparison, review, evaluation guide | Make a decision, compare options |
| Problem-solving | How-to, best practices, checklist | Solve a specific problem |
| Trend analysis | Thought leadership, data-driven | Understand what is changing and why |

## Phase 2: Outline Construction

### Post Architecture

Every blog post follows this skeleton:

```
[Meta Title] -- Primary keyword + compelling modifier (50-60 chars)
[Meta Description] -- Value prop + primary keyword (150-160 chars)

H1: [Post Title] -- Matches meta title or close variant

[Hook Intro] -- 2-3 paragraphs (150-200 words)
  - Opens with a pain point, question, or surprising data point
  - Establishes why this topic matters right now
  - Previews what the reader will learn

H2: [Section 1] -- Includes primary or secondary keyword
  [Content: 200-300 words]
  [Data point or example]

H2: [Section 2] -- Includes secondary keyword
  H3: [Subsection 2a]
  H3: [Subsection 2b]
  [Content: 300-400 words]

H2: [Section 3] -- Includes secondary keyword
  [Content: 200-300 words]
  [Expert quote placeholder or data citation]

H2: [Section 4] -- Practical application / how-to
  [Content: 200-300 words]
  [Actionable takeaway, checklist, or framework]

H2: [Conclusion / Key Takeaways]
  [Summary: 100-150 words]
  [CTA: Next step for the reader]

[Internal link suggestions]
[Author bio with E-E-A-T signals]
```

### Heading Optimization

- H1: One per post. Contains primary keyword. 50-70 characters.
- H2: 4-6 per post. Each contains a keyword variant. Scannable and descriptive.
- H3: Use under H2s when a section has 2+ sub-points. Not required for every section.
- Heading test: Someone scanning only headings should understand the post's argument.

## Phase 3: Writing

### Hook Introduction (First 150 Words)

The introduction must accomplish three things:
1. **Hook** -- Stop the reader from bouncing. Use a data point, question, scenario, or bold statement.
2. **Relevance** -- Make it clear who this is for and what they will gain.
3. **Preview** -- Tell them what the post covers without giving everything away.

**Hook templates for clinical trial audience:**

- **The Stat Hook:** "Clinical trials generate an average of [X] data points per study. When [Y]% require manual review, your data management team becomes a bottleneck. Here is how to change that."

- **The Question Hook:** "When was the last time your team finished a study build on schedule? If you are like most clinical data management teams, the answer involves a sigh and a story about protocol amendments."

- **The Scenario Hook:** "It is three weeks before your database lock deadline. Your query rate just spiked. The medical monitor has new edit check requests. Sound familiar?"

- **The Trend Hook:** "The FDA's latest guidance on decentralized clinical trials changes what your EDC platform needs to support. Here is what to prioritize."

### Body Content Rules

**Load brand-voice skill and apply throughout.**

1. **One idea per paragraph.** If a paragraph makes two points, split it.
2. **Evidence density.** Every 200-300 words should include a data point, example, or citation.
3. **Scannable.** Use bullet lists, numbered steps, bold key phrases, and short paragraphs.
4. **Clinical credibility.** Reference ICH guidelines, FDA guidance, industry benchmarks (Tufts CSDD), or published studies where relevant.
5. **Active voice.** "Data managers spend 40% of their time on queries" not "40% of data manager time is spent on queries."
6. **Specificity over generality.** "Reduce query rates from 12 per 100 pages to 4" beats "significantly reduce query rates."
7. **Natural keyword integration.** Primary keyword appears in first 100 words, at least one H2, and naturally 3-5 times in the body. Never force it.

### E-E-A-T Signal Integration

Google evaluates content on Experience, Expertise, Authoritativeness, and Trustworthiness. Embed these signals:

**Experience:**
- Reference real clinical trial workflows and scenarios
- Describe problems in language that shows firsthand understanding
- Include "we have seen" or "in our experience" where authentic
- Reference conversations with customers (anonymized)

**Expertise:**
- Cite specific regulations (21 CFR Part 11, ICH E6(R2), CDISC standards)
- Use correct industry terminology
- Reference published research or guidance documents
- Include technical depth appropriate for the audience

**Authoritativeness:**
- Author bio with relevant clinical trial / EDC experience
- Link to other authoritative Talosix content on the topic
- Reference industry organizations (DIA, SCDM, ACRP, CDISC)
- Include quotes from or references to recognized industry experts

**Trustworthiness:**
- Accurate regulatory claims (never overstate compliance)
- Balanced perspective (acknowledge complexity, tradeoffs)
- Clear attribution of data and claims
- Transparent about Talosix's role (acknowledge when we are making a product-adjacent point)
- Date published and last updated date

### Internal Linking Strategy

Every post should include 3-5 internal links:

1. **Contextual link** -- Links to a related blog post where the reader can go deeper on a sub-topic
2. **Product link** -- Links to a relevant Talosix product or feature page (max 1-2, not forced)
3. **Resource link** -- Links to a downloadable resource, white paper, or case study
4. **Category/pillar link** -- Links to the pillar page for this content cluster

**Link anchor text rules:**
- Use descriptive, keyword-rich anchor text
- Never use "click here" or "learn more" as standalone anchors
- Vary anchor text -- do not use the same phrase repeatedly
- Link naturally within the flow of the sentence

### Expert Quotes and Citations

Include 1-2 expert quote placeholders per post:

**Format:**
> "Quote text that adds credibility and a human perspective to the topic." -- [Name], [Title], [Organization]

**Options:**
- If a real, approved quote is available, use it with attribution
- If not, create a placeholder: "[Insert expert quote on the impact of risk-based monitoring on EDC requirements]"
- Alternatively, cite published statements from FDA, ICH, or industry leaders (with source)

## Phase 4: SEO Technical Elements

### Meta Title
- 50-60 characters
- Primary keyword near the beginning
- Include a modifier: "Guide," "Best Practices," "How to," year, or number
- Example: "Clinical Trial EDC Software: The Complete Evaluation Guide (2026)"

### Meta Description
- 150-160 characters
- Include primary keyword
- Include a value proposition (what the reader gains)
- Include a soft CTA or curiosity trigger
- Example: "Compare the top clinical trial EDC platforms on study build speed, compliance, and total cost. A data manager's guide to choosing the right system."

### Schema Markup Recommendations

Recommend the appropriate schema type for each post:

- **Article schema** -- Standard for all blog posts (headline, datePublished, author, publisher)
- **HowTo schema** -- For how-to or step-by-step posts
- **FAQ schema** -- For posts structured around common questions
- **BreadcrumbList** -- For navigational context

Provide the recommended schema type and key fields. Do not write full JSON-LD unless requested.

### Image and Media Guidance

- **Featured image:** Describe a concept for the hero image (what it should convey, not a literal description)
- **In-post visuals:** Recommend where charts, tables, screenshots, or diagrams would add value
- **Alt text:** Write descriptive alt text for every recommended image, including primary keyword where natural
- **File naming:** Suggest descriptive file names (e.g., "clinical-trial-edc-comparison-table.png")

After the post is written, provide this actionable distribution checklist:

### Pre-Publication
- [ ] Meta title and description set in CMS
- [ ] Featured image uploaded with alt text
- [ ] Internal links verified (no broken links)
- [ ] Schema markup implemented
- [ ] URL slug is clean and keyword-rich (e.g., /blog/clinical-trial-edc-evaluation-guide)
- [ ] Categories and tags assigned
- [ ] Author bio attached with headshot and credentials
- [ ] Open Graph and Twitter Card metadata set
- [ ] Mobile preview checked

### Publication Day
- [ ] Publish at optimal time (Tuesday-Thursday, 8-10 AM ET)
- [ ] Share on Talosix LinkedIn company page
- [ ] Share on Talosix Twitter/X account
- [ ] Post in relevant LinkedIn groups (if applicable)
- [ ] Send to email list segment (if high-value content)
- [ ] Notify sales team with a 2-sentence summary and link

### Post-Publication (Week 1-2)
- [ ] Monitor search console for indexing and initial impressions
- [ ] Repurpose into social media posts (use content-atomizer skill)
- [ ] Add to relevant email nurture sequences
- [ ] Internal link from 2-3 existing blog posts to the new post
- [ ] Share in industry Slack/Discord communities (if appropriate and allowed)

### Ongoing (Monthly)
- [ ] Check ranking position for primary keyword
- [ ] Update content if data or regulations change
- [ ] Refresh publish date if significant updates are made
- [ ] Monitor for backlink opportunities
